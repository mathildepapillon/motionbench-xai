# MotionBench-XAI — Multi-Agent Build Plan

> **For:** Cursor 3.x with parallel agents (Sonnet 4.6 default model).
> **Source repos:** Forked CARE-PD codebase (overloaded WIP) + your existing MotionSHAP draft codebase.
> **Target:** New from-scratch repo `motionbench-xai/` structured as a NeurIPS Datasets & Benchmarks submission.
> **Submission:** NeurIPS 2026 D&B track.

---

## 0. How to use this document

This plan is meant to be loaded into Cursor as the project's authoritative spec. It is structured so that:

1. **Phase 0** (you, manually, before launching agents) locks the *contracts* — the four base interfaces every agent will plug into. Once these are merged to `main`, parallel work cannot collide on type signatures.
2. **Each later phase contains independent worktree-friendly tasks** — one agent per task, each in its own git worktree (`git worktree add ../mbxai-task-NN -b task/NN-name`). Tasks within a phase can run in parallel; phases are mostly sequential because later phases depend on earlier types.
3. **A shared `TASKS.md` file** (created in Phase 0) acts as the agent ledger — every agent reads and updates it before/after its task. This is the single source of truth for what is done, in progress, or blocked. Locking is by claiming a task row, not by file locks.
4. **Every agent is given the same system prompt** (`AGENTS.md` at repo root). Per-task instructions go in `tasks/TASK_NN.md` files generated by you from this plan.

**Operating discipline for all agents (must appear in `AGENTS.md`):**

- Before starting: read `TASKS.md`, claim your row by setting `status: in-progress` and `agent: <worktree-name>`.
- Read the **entire** task spec file (`tasks/TASK_NN.md`) and the relevant **base interface file** (`motionbench/<module>/base.py`) before writing any code.
- Do not modify base interfaces during your task. If you believe an interface is wrong, stop and write your concern into `TASKS.md` under the task row; do not silently change.
- Use existing libraries (Captum, Quantus, shap, tsCaptum) wherever possible. Implement from scratch only when the task spec explicitly says so.
- Every file you write must have: module docstring with one-line summary, full type hints, Google-style docstrings on public functions, and at least one unit test in `tests/`.
- After finishing: run `pytest tests/<your-module>/` and `ruff check .` and `mypy motionbench/`. Mark task `status: done` and write a 3-line summary in the task row's `notes` field.
- If you discover unrelated bugs or improvements, log them in `BACKLOG.md` instead of fixing — do not scope-creep.

**Cursor-specific operating notes:**

- Use `git worktree add` for every task (Cursor's default in v3+ Agents Window).
- For tasks marked **[needs thinking]** below, run with Sonnet 4.6 in extended-thinking mode. For mechanical port tasks marked **[mechanical]**, default Sonnet 4.6 is fine.
- For tasks marked **[verify against literature]**, the agent must use web search and cite sources in code comments and the PR description.

## 2. External dependencies (do not reimplement)

The plan below assumes these libraries. Pin versions in `pyproject.toml`.


| Library                         | Used for                                                            | Why this one                                              |
| ------------------------------- | ------------------------------------------------------------------- | --------------------------------------------------------- |
| `captum`                        | IG, DeepLift, GradientShap, Saliency, SmoothGrad, GradCAM           | de facto standard, PyTorch-native                         |
| `shap`                          | KernelExplainer (we override the masker with our pluggable imputer) | reference implementation                                  |
| `quantus`                       | Fidelity, stability, sanity-check metrics                           | JMLR-published, time-series support, batch implementation |
| `zennit`                        | LRP for arbitrary PyTorch models                                    | best-maintained LRP impl                                  |
| `tscaptum`                      | Time-series segmentation wrapper around Captum                      | handles long sequences                                    |
| `timeshap`                      | TimeSHAP (Bento et al.)                                             | reference implementation                                  |
| `windowshap`                    | WindowSHAP (Nayebi et al.)                                          | reference implementation                                  |
| `shats`                         | ShaTS (Lopez et al.)                                                | reference implementation                                  |
| `aeon` / `sktime`               | Time-series classifier baselines                                    | for synthetic sanity checks                               |
| `pyro` or `torch` distributions | Gaussian/copula sampling                                            | already in PyTorch                                        |
| `pyvinecopulib`                 | Vine copula imputer baseline                                        | only mature option                                        |
| `wandb`                         | Experiment tracking                                                 | standard for D&B                                          |
| `hydra-core`                    | Config composition                                                  | standard for D&B                                          |


**Hard rule:** if a library above implements what a task asks for, the task wraps it. Do not re-derive Shapley kernels, IG paths, or LRP rules from scratch. The contribution is the *unified evaluation framework*, not the underlying methods.

### 0.2 Lock the four base abstractions

These are the only files agents are forbidden to modify. Write them by hand. They go in:

- `motionbench/data/base.py`
- `motionbench/oracles/base.py`
- `motionbench/imputers/base.py`
- `motionbench/attribution/base.py`
- `motionbench/metrics/base.py`
- `motionbench/players/base.py`

Each base file contains: ABC class with all required methods declared, full type hints, exhaustive docstrings, **and a `tests/test_<base>_contract.py` that asserts any concrete subclass conforms** (ABC won't catch shape mismatches; write a contract test that calls every method on a `MockSubclass`).

Minimum signatures (Sonnet 4.6 should expand these):

```python
# data/base.py
class BaseDataset(Protocol):
    def __getitem__(self, idx: int) -> tuple[Tensor, Tensor]: ...   # (x, y)
    def __len__(self) -> int: ...
    @property
    def shape(self) -> tuple[int, ...]: ...                          # (J, F, T) for motion
    @property
    def metadata(self) -> dict: ...                                  # skeleton, frame rate
    @property
    def oracle(self) -> Optional["Oracle"]: ...                      # None for real data

class GroundTruthDataset(BaseDataset, Protocol):
    @property
    def oracle(self) -> "Oracle": ...                                # required, not Optional

# oracles/base.py
class Oracle(ABC):
    @abstractmethod
    def conditional_sample(self, x_obs: Tensor, mask: Tensor, n: int) -> Tensor: ...
    @abstractmethod
    def true_shapley(self, x: Tensor, classifier: Callable, players: "PlayerSet") -> Tensor: ...

# imputers/base.py
class BaseImputer(ABC):
    @abstractmethod
    def fit(self, train_data: BaseDataset) -> "BaseImputer": ...
    @abstractmethod
    def impute(self, x_obs: Tensor, mask: Tensor, n_samples: int) -> Tensor: ...
    # Same contract as Oracle.conditional_sample — Oracle is essentially a perfect Imputer.

# attribution/base.py
class BaseAttributor(ABC):
    def __init__(self, classifier: Callable, **kwargs): ...
    @abstractmethod
    def attribute(self, x: Tensor, players: "PlayerSet", target: int) -> Tensor: ...
    # KernelShapAttributor accepts an `imputer: BaseImputer`; gradient methods don't.

# metrics/base.py
class BaseMetric(ABC):
    requires_oracle: ClassVar[bool] = False
    @abstractmethod
    def evaluate(self, phi: Tensor, x: Tensor, classifier: Callable,
                 oracle: Optional[Oracle] = None,
                 imputer: Optional[BaseImputer] = None) -> dict[str, float]: ...
```

## 4. Phase 1 — Synthetic data and oracles

**Goal:** all synthetic generators + closed-form oracles ported and validated. **Parallelism:** 4 agents.

### Task 1A — Port and extend Gaussian Motion generator [needs thinking]

**Worktree:** `task/1A-gaussian-motion`
**Depends on:** Phase 0
**Files:** `motionbench/data/synthetic/gaussian_motion.py`, `motionbench/oracles/gaussian_oracle.py`, `tests/test_gaussian_oracle.py`

**Inputs from old code:** `motionshap_draft/synthetic/gaussian.py` (per `SOURCE_MAP.md`).

**Spec:**

1. Port the existing equicorrelated-Σ_joint, AR(1)-Σ_time generator from the draft.
2. Add structured Σ_joint variants:
  - `equicorrelated(rho)` — current default
  - `skeleton_adjacency(skeleton="h36m", decay=0.7)` — Σ_joint built from Human3.6M kinematic tree, off-diagonal = `decay ** graph_distance`
  - `block_diagonal(left_right=True)` — left and right body blocks
  - `data_driven(care_pd_subset)` — empirical Σ_joint estimated from a CARE-PD subset (cache the estimate to disk)
3. Add structured Σ_time variants: AR(1), AR(p), gait-periodic Toeplitz with controllable period (use a sum-of-cosines kernel to inject periodicity).
4. Implement `GaussianOracle` in `oracles/gaussian_oracle.py`:
  - `conditional_sample`: closed-form via the standard Gaussian conditional formula μ_{a|b} = μ_a + Σ_ab Σ_bb^{-1} (x_b − μ_b).
  - `true_shapley`: enumerate all 2^n coalitions when n ≤ 12; otherwise raise NotImplementedError (your synthetic K and J values stay small).
5. Tests:
  - Unit test that `conditional_sample` matches large-N rejection sampling within tolerance for a random small Σ.
  - Unit test that `true_shapley` satisfies the efficiency axiom (Σ ϕ_i = v(N) − v(∅)) to numerical precision.
  - Unit test with diagonal Σ that on-manifold and marginal imputation give identical Shapley values (sanity check).

**Best-practice references the agent must consult:**

- Aas et al. (2021) "Explaining individual predictions when features are dependent" — derivation of conditional Gaussian for KernelSHAP. Cite in docstring.
- `shap` library's `MaskedModel` for the canonical Kernel SHAP estimator interface.

**Output:** PR titled `[1A] Gaussian motion + closed-form oracle` with all tests green.

### Task 1B — Port Burr / Gaussian-copula generator [needs thinking]

**Worktree:** `task/1B-burr-motion`
**Depends on:** Phase 0
**Files:** `motionbench/data/synthetic/burr_motion.py`, `motionbench/oracles/copula_oracle.py`, `tests/test_copula_oracle.py`

**Inputs from old code:** `motionshap_draft/synthetic/burr.py`.

**Spec:**

1. Port the Burr XII Gaussian-copula generator from the draft (heavy-tailed marginals, Gaussian dependence).
2. Generalize: support multiple marginal families via a clean strategy pattern (`Marginal` ABC with `cdf`, `quantile`, `pdf`):
  - `BurrXII(c, k)` — current
  - `StudentT(df)` — parametric tail control
  - `MixtureOfGaussians(weights, means, scales)` — multimodal motion clusters
  - `SkewNormal(alpha)` — asymmetric, mimics real motion better than symmetric Burr
3. Implement `CopulaOracle` that takes a Σ (joint × time Kronecker) and a per-coordinate `Marginal`:
  - `conditional_sample`: transform observed coordinates to latent Gaussian via Φ⁻¹(F_i(x_i)), apply Gaussian conditional, transform back via F_i⁻¹(Φ(z_i)).
  - `true_shapley`: same enumeration as Gaussian case.
4. Tests:
  - Round-trip test: marginal CDF→quantile→CDF preserves values to numerical precision for each `Marginal`.
  - Test that with Gaussian marginals, `CopulaOracle` and `GaussianOracle` give numerically identical Shapley values (≤ 1e-5).
  - Test efficiency axiom on Burr.

**Best-practice references:**

- Joe (2014) *Dependence Modeling with Copulas* — copula transform identity.
- Aas et al. (2021), Chapter on copula-based conditional expectation in SHAP.
- `pyvinecopulib` source for reference Gaussian copula sampling.

**Output:** PR titled `[1B] Burr/copula generator + oracle`.

**Depends on:** Phase 0
**Files:** `motionbench/imputers/off_manifold.py`, `tests/test_off_manifold_imputers.py`

**Spec:** Implement four trivial imputers conforming to `BaseImputer`:

2. `MeanImputer` — fill with per-coordinate training mean (computed in `.fit`).
3. `MarginalDonorImputer` — fill missing coordinates by drawing from a random training sequence (canonical "marginal SHAP").
4. `GaussianNoiseImputer(scale)` — fill with mean ± Gaussian noise.

Each imputer is ~30–60 lines. Tests verify shapes, that observed coordinates are preserved, and that `n_samples` returns the requested batch size.

**Output:** PR titled `[2A] Off-manifold imputers`.

### Task 2E — KernelSHAP attributor wrapping `shap` library [needs thinking]

**Worktree:** `task/2E-kernelshap`
**Depends on:** 2A (needs at least one imputer to test against)
**Files:** `motionbench/attribution/kernel_shap.py`, `tests/test_kernel_shap.py`

**Spec:** This is the linchpin task. Build a `KernelShapAttributor(BaseAttributor)` that:

1. Accepts a `BaseImputer` and a `PlayerSet` at construction.
2. Internally constructs a custom `shap.maskers.Masker` whose `__call__(mask, x)` defers to `imputer.impute(x_obs, mask, n_samples)`. This is how `shap` officially supports custom maskers — read `shap.maskers.Independent` source as the reference.
3. Calls `shap.KernelExplainer` (or `shap.Explainer` with `algorithm="permutation"` if it's faster on your sequence sizes — benchmark and pick).
4. Returns Shapley values *aggregated to the player level* — the agent must implement the reduction from per-coordinate Shapley to per-player Shapley using the additivity of Shapley values across coalitions of indistinguishable elements.

Tests:

- With `GaussianOracle` as the imputer (oracle implements `BaseImputer` trivially), KernelShapAttributor matches the oracle's `true_shapley` to within Monte Carlo tolerance.
- Compare wall-clock against a from-scratch implementation to verify wrapping `shap` is at least as fast.

**This task must use the existing `shap` library.** Do not re-derive Kernel SHAP weights or solve the weighted linear regression by hand. The contribution is the masker.

**Output:** PR titled `[2E] KernelSHAP with pluggable imputer`.

## 11. Risk register

These are the things that can sink the paper. Keep them in `BACKLOG.md` and revisit weekly.

1. **CARE-PD encoder reproducibility (Task 4B).** If checkpoints don't reproduce CARE-PD's reported F1, the real-data section is wobbly. Mitigation: contact CARE-PD authors early; have a fallback plan to use only the encoders you can reproduce.
2. **The M=10 Burr Flow regression (Task 2D).** Either fixed or honestly characterized — never silent.
3. **Quantus's `perturb_func` integration (Task 5B).** If Quantus's API doesn't cleanly accept arbitrary imputers, you'll need to fork or vendor — adds days. Mitigation: spike this within first week of Phase 5, before committing.
4. `**shap.KernelExplainer` with custom masker performance (Task 2E).** May be too slow on long sequences. Mitigation: switch to `shap.Explainer(algorithm="permutation")` or implement segmented Kernel SHAP via `tscaptum`.
5. **Scope creep.** This plan covers ~3 weeks of focused work. Anything not in this plan goes in `BACKLOG.md`, not `TASKS.md`.
