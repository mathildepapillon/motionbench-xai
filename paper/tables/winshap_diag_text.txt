WindowSHAP window-size diagnostic: synthetic\_transformer, gaussian\_k4 and skeleton\_gait\_combined (T=16, K=4), 50 sequences each.

The smallest window ($w{=}2$) achieves the lowest EC1 on both datasets (0.236 on \texttt{gauss\_k4}; 0.672 on \texttt{skel+gait}), while EC1 grows monotonically with $w$ and reaches its worst value at $w{=}15$ (${\approx}T{-}1$), attaining 2.648 and 1.585 respectively—more than ten times the KS-VAEAC reference of ${\approx}0.016$ on either dataset.
WindowSHAP fails for a structural reason: its sliding-window kernel assigns SHAP values to \emph{overlapping} windows and then aggregates per-timestep attributions via a running nanmean; this aggregation averages contributions from windows that include and exclude each frame, violating the boundary-coalition constraint enforced by KernelSHAP, which requires each coalition to be a binary subset of the $K$ defined players—the result is magnitude-shifted attributions whose absolute scale grows with $w$ as more overlapping windows contribute conflicting partial SHAP values to each timestep.
The degradation with $w$ is steep for \texttt{gauss\_k4} (EC1 increases 11× from $w{=}2$ to $w{=}15$), and somewhat flatter for \texttt{skel+gait} (2.4× increase), likely because the skeleton–gait dataset's richer spatial covariance structure spreads oracle attribution mass across many joints, softening the per-player magnitude error even as the temporal-aggregation bias remains severe.
KS-VAEAC, which uses a proper conditional-expectation imputer and respects the $K$-player coalition structure, achieves EC1~${\approx}0.016$ on both datasets at any classifier—roughly $14\times$ better than the best WindowSHAP configuration ($w{=}2$) and more than $160\times$ better than the worst ($w{=}15$), confirming that correct imputation and coalition semantics are the dominant factors, not window granularity.
